# 匿名 AI 交友匹配 Demo 需求-规约文档 v1.0

文档用途：指导 Codex 完成网页 Demo 的产品设计与前端开发。  
交付类型：移动端优先 Web Demo / H5 感 Web App。  
当前版本：v1.0。  
默认城市：北京。  
默认活动场景：咖啡、咖啡聊天、咖啡店周边轻散步。  
目标语言：中文。  
用户范围：成人用户。  
头像方式：抽象头像，不展示真实照片。  
好友申请：Demo 默认自动通过。  
AI 能力：当前全部使用预设脚本和 mock 数据伪造，无需接入真实 AI。

---

## 0. 给 Codex 的一键开发摘要

请开发一个移动端优先的匿名 AI 交友匹配 Web Demo。技术栈建议使用 Vite + React + TypeScript + Tailwind CSS。无需后端、无需真实 AI、无需真实数据库。所有 AI 对话、画像、匹配结果、推荐理由、聊天、好友申请、活动确认均使用 mock 数据和本地状态模拟。

用户路径为：进入 Landing Page -> 开始 AI 预设对话建模 -> 生成匿名社交画像 -> 进入首页 -> 体验三种匹配方案：1v1 咖啡活动匹配、多 v 多咖啡活动匹配、AI 匿名卡片推荐。好友申请默认自动通过。整体 UI 要更偏移动端 App 感，中文界面，浅色、温和、匿名、可信、低压力社交风格。

开发重点不是算法，而是完整演示闭环：AI onboarding 可走完；首页能看到三类匹配；1v1 可完成同意匹配、自动成为好友、活动确认和集合地点沟通；多人活动可加入群聊、查看 AI 地点推荐、进行地点投票并确认；卡片推荐可发送好友申请并自动通过；好友页能看到新增好友和群聊；我的画像页可编辑 DIY 匿名卡片。

## 1. 项目背景与目标

### 1.1 产品概念

本项目是一个匿名交友网页 Demo，用于展示“AI 先与用户对话进行人像建模，再基于用户画像进行 CP / 活动 / 好友推荐”的核心概念。当前阶段不需要真实 AI，只需要通过预设对话、预设画像、预设推荐和本地交互状态完成产品体验演示。

### 1.2 Demo 目标

- 让用户在 2 分钟内理解：本产品不是传统刷照片交友，而是先由 AI 理解用户，再提供少量、高质量、低压力的社交连接。
- 展示三种核心匹配路径：1v1 活动匹配、多 v 多活动匹配、AI 卡片推荐。
- 展示匿名性、安全感和活动驱动型交友机制。
- 为投资人、团队成员、设计师或开发者提供可点击、可演示的前端原型。

### 1.3 当前非目标

当前阶段不实现以下能力：真实 AI 对话、真实画像算法、真实匹配算法、真实用户注册登录、真实即时聊天、真实地图定位、真实支付、真实 Push 通知、实名认证、照片上传、后台管理系统、内容审核后台。

## 2. 已确认产品约束

### 2.1 基础约束

- 城市默认为北京。
- 场景偏向咖啡，包括咖啡聊天、咖啡店见面、咖啡店周边轻散步。
- 用户均为成人，界面可展示“仅限成人用户”的轻提示。
- Demo 中文界面。
- 所有用户头像均为抽象头像，不使用真实人像照片。
- 好友申请默认自动通过，便于演示完整闭环。
- 产品更偏移动端，应按手机屏幕优先设计，再适配桌面端。

### 2.2 Demo 边界

- 不需要调用 meetonce.ai 或任何外部服务。
- 不需要后端接口。
- 不需要数据库。
- 不需要用户真实登录。
- 所有状态可存在 React state，并用 localStorage 保留核心进度。
- 所有推荐、聊天、AI 理由、活动地点均为 mock 数据。

### 2.3 安全与匿名原则

页面中需要体现以下规则：不展示真实姓名、手机号、微信号、详细住址；首次见面建议选择公共场所；提供“举报 / 屏蔽 / 不感兴趣”入口，但 Demo 中点击后只显示提示；默认展示匿名昵称和抽象头像。

## 5. 核心用户旅程

### 5.1 首次访问旅程

```txt
Landing Page
-> 点击“开始 AI 建模”
-> 进入预设 AI 对话
-> 用户选择若干偏好选项
-> 右侧或下方画像标签逐步生成
-> 完成匿名画像
-> 点击“进入我的匹配”
-> 首页看到三种匹配入口
```

### 5.2 1v1 咖啡匹配旅程

```txt
首页点击“1v1 咖啡活动匹配”
-> 查看匿名对象、AI 推荐理由、咖啡活动建议
-> 点击“同意本次匹配”
-> 显示等待对方确认
-> 1-2 秒后模拟对方同意
-> 自动添加好友
-> 展示活动确认聊天窗口
-> 发送集合地点建议
-> 点击“确认活动”
-> 状态变为“活动已确认”
```

### 5.3 多 v 多咖啡局旅程

```txt
首页点击“多 v 多咖啡局”
-> 查看 3v3 或 6v6 活动主题、成员、AI 地点推荐
-> 点击“加入本周活动”
-> 自动进入群聊
-> 查看 AI 推荐流程与地点投票
-> 选择一个北京咖啡区域
-> 点击“确认加入该安排”
-> 状态变为“已确认”
```

### 5.4 卡片推荐旅程

```txt
首页点击“AI 卡片推荐”
-> 浏览匿名推荐卡片
-> 点击“发送好友申请”
-> 1-2 秒后自动通过
-> 该用户进入好友列表
-> 可在好友页聊天
```

## 6. UI 设计规范

### 6.1 设计关键词

```txt
移动端优先
温和
匿名
可信
低压力
咖啡感
轻社交
AI 辅助但不过度科技化
```

### 6.2 视觉方向

- 使用浅色背景、大圆角卡片、柔和渐变、轻阴影。
- 抽象头像可以使用几何图形、渐变圆、emoji 风格符号，但不要使用真实照片。
- 主视觉建议偏奶油白、浅咖啡色、柔和蓝紫、低饱和橙色。
- 匹配度、状态、标签使用 chip / badge 展示。
- 所有页面都要避免信息密度过高，优先符合手机浏览。

### 6.3 建议色彩

```txt
背景色：#F8F5EF / #FAF7F2
主色：#6F5EF9 或 #7C6CF2
咖啡辅助色：#B9825B / #D8B08C
成功色：#3A8F6A
警示色：#C77D2D
正文：#232323
次级文字：#6F6A64
卡片：#FFFFFF
边框：#ECE2D8
```

### 6.4 移动端布局要求

- 首屏宽度按 375px - 430px 设计。
- 底部导航高度不超过 72px。
- 主按钮高度建议 44px - 52px。
- 卡片圆角建议 20px - 28px。
- 页面左右 padding 建议 16px。
- 聊天输入框固定在聊天区底部，避免横向溢出。
- 重要 CTA 使用 sticky bottom 区域或卡片底部固定按钮。

### 7.2 AI 对话建模页 /onboarding

页面目标：模拟 AI 与用户对话，逐步生成人像 / 用户画像。

移动端布局：

```txt
顶部：进度条 + 标题“AI 正在了解你”

下部：用户可选回复按钮 / 输入框
底部或折叠卡：画像生成结果、已识别标签、匿名卡片预览
```

预设聊天步骤：

Step 1：

```txt
AI：你好，我会通过几个轻松的问题了解你的社交偏好。你希望别人第一次认识你时，先感受到你哪一面？
用户选项：温和可靠 / 有趣松弛 / 独立清醒 / 好奇心强
新增标签：温和可靠 / 有趣松弛 / 独立清醒 / 好奇心强
```

Step 2：

```txt
AI：如果这周在北京安排一次见面，你更期待哪种咖啡场景？
用户选项：安静咖啡聊天 / 咖啡 + 轻散步 / 精品咖啡探店 / 商圈咖啡快闪 / 书店咖啡
新增标签：咖啡聊天 / 轻散步 / 探店 / 商圈见面 / 书店咖啡
```

Step 3：

```txt
AI：你更喜欢怎样的聊天节奏？
用户选项：慢慢熟悉，不急着交换隐私 / 可以轻松开玩笑 / 喜欢深入聊价值观 / 希望对方主动一点
新增标签：慢热 / 幽默感 / 深度聊天 / 偏好主动型搭子
```

Step 4：

```txt
AI：第一次咖啡见面时，你希望 AI 帮你避开什么样的匹配？
用户选项：目的性太强 / 只看照片 / 聊天压力太大 / 临时爽约 / 过早索要联系方式
新增标签：边界感 / 低压力社交 / 反爽约 / 隐私保护
```

Step 5：

```txt
AI：我已经生成了你的匿名社交画像。接下来可以基于这个画像，为你推荐 1v1 咖啡活动、多人数咖啡局和用户卡片。
```

```txt
推荐匹配方式：1v1 咖啡活动、3v3 咖啡轻社交局、少量高质量卡片推荐
```

### 7.4 1v1 咖啡活动匹配 /match/one-on-one

页面目标：展示每人每周一次的 1v1 咖啡活动报名与双方同意后的好友添加、活动确认流程。

状态机：

```txt
recommended -> waiting_other -> activity_confirming -> confirmed
```

状态说明：

```txt
recommended：AI 已推荐，等待当前用户同意
waiting_other：当前用户已同意，等待对方同意
activity_confirming：双方已同意，自动添加好友，进入活动确认阶段
confirmed：活动已确认
```

Demo 简化：点击“同意本次匹配”后，等待 1-2 秒自动模拟对方同意，然后进入 activity_confirming。

主卡片内容：

```txt
本周 1v1 咖啡匹配
匹配对象：胡同回声
匿名头像：抽象头像
匹配度：91%
活动建议：周六下午咖啡聊天
时间建议：本周六 15:00 - 16:30
地点建议：北京三里屯 / 亮马河 / 五道营附近咖啡店
当前状态：等待你确认
```

对象卡片：

```txt
匿名昵称：胡同回声
年龄段：25-29
城市：北京
AI 标签：安静幽默、喜欢咖啡、周末可约、边界感强
一句话卡片：比起热闹局，更喜欢一杯咖啡里慢慢展开的话题。
```

AI 推荐理由：

```txt
你们都偏好低压力的咖啡见面，也都在对话中表达了对边界感和稳定节奏的重视。相比高强度社交，你们更可能在安静咖啡店中自然熟悉，并通过轻松话题逐步建立信任。
```

活动安排卡：

```txt
活动主题：咖啡聊天
建议时长：60-90 分钟
建议预算：人均 40-80 元
建议区域：三里屯、亮马河、五道营、国贸商圈
安全建议：请选择公共场所，首次见面不建议透露住址或私人联系方式。
```

```txt

暂时跳过
查看 AI 推荐理由
```

点击“同意本次匹配”后：

```txt

1-2 秒后 toast：对方也已同意，已自动添加为好友
状态进入 activity_confirming
页面展开集合地点沟通聊天窗口
```

集合地点沟通聊天：

```txt
系统：你们已互相同意，已自动添加为好友。请确认咖啡时间与集合地点。
胡同回声：你好，我觉得三里屯附近可以，咖啡选择比较多。
```

确认后：

- 用户能在 2 分钟内理解产品核心机制。

- 不应要求用户登录。
- 不应请求真实定位权限。
- 不应上传真实照片。
- 不应调用真实 AI API。
- 不应依赖后端服务。
